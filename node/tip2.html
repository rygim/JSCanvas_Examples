Always do the least amount possible

Interpolation

  • Expensive update operations can possibly be delayed (Game AI, State Machine, etc.)
  • Can still interpolate the positions of the new objects without the bulk of processing going on
Interpolation Example
Update25ms
Draw+15ms
Interpolation+15ms
UI Thread Update+1ms
Total Time=51ms/frame = 1frame/51ms * 1000ms = 19.6fps best case

Web Workers

  • Javascript can now execute threads
  • The API is in beta, and not part of canvas
  • The threads can't make UI changes, similar to the UI thread on android devices

Other concerns

  • jQuery, underscore, prototype are useful, but they add a lot of overhead and kill performance
  • It takes a non-negligible amount of time to access array items and members in the prototype chain
  • Caching local variables in memory may provide a good speed boost
